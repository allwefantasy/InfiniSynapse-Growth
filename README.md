# InfiniSynapse-Growth 运营存档

本仓库用于归档 InfiniSynapse 运营团队对外发布的所有内容，确保每一份对外输出都有源可查、有迹可循。

---

## 目录结构

```
InfiniSynapse-Growth/
├── 日常运营/
│   └── YYYY-MM-DD/
│       └── <GitHub 用户名>/
│           └── ...(具体内容)
├── 活动/
│   └── <活动名称>/
│       └── ...(具体内容)
├── 品牌素材/
│   └── ...(Logo、VI、模板等)
├── 长期文档/
│   └── ...(白皮书、FAQ、介绍等)
└── 外部合作/
    └── ...(合作方相关材料)
```

### 1. 日常运营

存放日常对外发布的内容(如日报、推文、评论、社群消息、公告等)。

- **第一层级**：日期目录，格式为 `YYYY-MM-DD`(例如 `2026-04-21`)
- **第二层级**：负责人的 GitHub 用户名(例如 `alice`、`bob`)
- **第三层级**:该负责人当天发布的具体内容文件或子目录

**示例**：

```
日常运营/
├── 2026-04-21/
│   ├── alice/
│   │   ├── twitter-post.md
│   │   └── discord-announcement.md
│   └── bob/
│       └── weekly-digest.md
└── 2026-04-22/
    └── alice/
        └── blog-update.md
```

### 2. 活动

存放运营活动相关的所有对外材料(如 AMA、黑客松、空投、联合活动等)。

- **第一层级**：活动名称目录(建议使用 `YYYY-MM-活动名` 格式便于排序)
- **内部结构**：由活动负责人自行组织(可包含公告文案、海报、规则说明、复盘文档等)

**示例**：

```
活动/
├── 2026-04-Hackathon/
│   ├── 公告.md
│   ├── 规则.md
│   └── 海报/
└── 2026-05-AMA/
    ├── 预告.md
    └── 问答整理.md
```

### 3. 品牌素材

存放对外使用的品牌相关资源，方便复用与统一风格。

- 适用内容：Logo、VI 规范、配色、字体、头像、Banner、PPT/文档模板、宣传图模板等
- 建议按类型分子目录(如 `Logo/`、`模板/`、`配色/`)

**示例**：

```
品牌素材/
├── Logo/
│   ├── logo-light.png
│   └── logo-dark.png
├── VI 规范.pdf
├── 模板/
│   ├── 推文封面.psd
│   └── 公告海报.fig
└── 配色与字体.md
```

### 4. 长期文档

存放长期对外有效、需要持续维护的文档内容。

- 适用内容：项目介绍、白皮书、产品 FAQ、Roadmap 对外版、官方话术、常见问答、媒体包(Media Kit)等
- 建议保留更新记录(可在文件顶部或底部注明最后更新日期与负责人)

**示例**：

```
长期文档/
├── 项目介绍.md
├── 白皮书 v1.2.pdf
├── FAQ.md
├── Media-Kit/
│   ├── 一句话介绍.md
│   └── 产品截图/
└── 官方话术.md
```

### 5. 外部合作

存放与外部合作方相关的对外材料。

- **第一层级**：合作方名称目录(例如 `Partner-A`、`Media-B`)
- **内部结构**：该合作相关的公告、联合推文、物料、合同附件(非机密部分)等

**示例**：

```
外部合作/
├── Partner-A/
│   ├── 联合公告.md
│   ├── 合作推文.md
│   └── 物料/
└── Media-B/
    └── 采访稿.md
```

## 使用规范

1. **所有对外发布前先归档**：运营在对外发送任何内容(推文、公告、文案、图片等)之前，先在本仓库对应目录下创建文件并提交。
2. **文件命名清晰**：建议使用 `渠道-主题.md` 或 `序号-主题.md` 的命名方式,便于查找。
3. **保留原始素材**：涉及图片、视频、设计稿等非文本素材,请一并存放到对应目录的子文件夹中。
4. **提交信息规范**：Commit message 建议格式为 `[顶层目录] 日期 或 名称 - 简述`,例如：
   - `[日常运营] 2026-04-21 alice - 发布产品更新推文`
   - `[活动] 2026-04-Hackathon - 发布报名公告`
   - `[品牌素材] 更新暗色版 Logo`
   - `[长期文档] 更新 FAQ - 新增 Q7`
   - `[外部合作] Partner-A - 发布联合公告`
5. **敏感信息**：切勿在仓库中提交密码、私钥、内部 API Key 等敏感信息。

## 快速开始

```bash
git clone <本仓库地址>
cd InfiniSynapse-Growth

mkdir -p "日常运营/$(date +%Y-%m-%d)/<你的 GitHub 用户名>"

git add .
git commit -m "[日常运营] $(date +%Y-%m-%d) <你的 GitHub 用户名> - <简述>"
git push
```
