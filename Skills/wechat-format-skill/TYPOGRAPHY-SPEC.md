# 微信公众号排版执行规范（纽约时报风格）

| 主题 ID | 视觉特征 | 适合内容 |
|---------|---------|---------|
| `newspaper` | 纽约时报衬线、双线标题、严肃克制 | 深度长文、行业分析、评论 |

## 二、格式规则清单

### 📌 1. 标题层级

| 层级 | 规格（newspaper 主题） |
|------|-----------------------|
| **H1** | 29px 加粗，上下双线分隔（#111），左对齐 |
| **H2** | 19px 大写字间距（0.12em），居中，上下细线 |
| **H3** | 17px 加粗，左对齐 |
| **H4** | 15px 加粗 |
| **H5** | 14px 加粗 |
| **H6** | 13px 灰色（#666） |

⚠️ **重要**：正文首个 H1 会被自动移除（公众号编辑器顶部已有独立标题输入框）

---

### 📌 2. 段落规范
- 字号：15px
- 行高：1.7
- 字间距：0.5px
- 段间距：18px（margin-bottom）
- 对齐：左对齐
- 字体：Georgia / 思源宋体（衬线优先）

---

### 📌 3. 引言 / 引用规范

#### 普通引用（Blockquote）
- 背景色：米色 #f7f3ee
- 边框：上下各 1px #d5cfc5
- 内边距：18px 上下，20px 左右
- 文字：16px 斜体，灰色 #555

#### Callout 强调块（4种类型）
| 类型 | Markdown 标记 | 左边界颜色 | 背景色 |
|------|--------------|-----------|-------|
| 重要观点 | `> [!important] 标题` | #326891 | 渐变蓝 |
| 实战技巧 | `> [!tip] 标题` | #22c55e | 浅绿 |
| 注意事项 | `> [!warning] 标题` | #f59e0b | 浅橙 |
| 普通强调 | `> [!callout]` | 主题色 | 浅灰 |

---

### 📌 4. 图片规范
- 布局：独占一行 + 居中
- 圆角：4px
- 阴影：0 2px 8px rgba(0,0,0,0.08)
- 最大宽度：100%

#### 图说（Caption）
- 紧跟图片下方
- 居中、灰色（#888）、13px、斜体

#### 特殊容器
- **Gallery 横向滚动**：3 张以上连续图片自动应用
- **Long Image 纵向滚动**：长图固定 600px 高度

---

### 📌 5. 分隔线（HR）规范
- 渐变细线：从透明 → 黑色 → 透明
- 宽度：38%
- 居中显示
- 上下边距：40px

---

### 📌 6. 强调方式

#### 加粗（Strong）
- 黑色文字 + 蓝色线性渐变高亮（模拟划重点效果）
- 高亮范围：文字底部 38% 区域

#### 斜体（Emphasis）
- 灰色 #666
- 标准斜体

---

### 📌 7. 结尾 CTA / 参考链接
- 顶部 1px 灰色分割线
- 标题：13px 灰色加粗
- 链接列表：13px 灰色，行高 1.6
- 格式：`[1] 标签: URL`

---

### 📌 8. 对话气泡（Dialogue）
- 容器：浅灰背景，8px 圆角，16px 内边距
- 左右交替布局
- 左侧（第一位发言者）：白底，圆角 12px 12px 12px 0
- 右侧（第二位发言者）：浅橙底，圆角 12px 12px 0 12px
- Markdown 格式：
```markdown
:::dialogue[嘉宾对谈]
张三: 你怎么看这件事？
李四: 我觉得...
:::
```

## 三、纽约时报风格落地细节

### 配色方案
| 用途 | 色值 |
|------|------|
| **主色（Primary）** | `#326891`（冷蓝灰） |
| **正文（Text）** | `#111111`（纯黑） |
| **弱化文字（Muted）** | `#666666` |
| **背景** | `#ffffff`（纯白） |
| **引用背景** | `#f7f3ee`（米色） |
| **代码块背景** | `#f0ede8` |
| **列表圆点** | `#326891` |

### 字体策略
```
Georgia → 'Source Han Serif SC' → 'Noto Serif CJK SC' → serif
```
- 英文优先使用 Georgia（纽约时报正文字体）
- 中文回退到思源宋体
- 始终使用衬线字体，营造报纸印刷感

### 视觉特征
1. ✅ 克制严肃，无花哨装饰
2. ✅ 衬线字体 + 合适的字间距/行高
3. ✅ 双线 H1 + 居中大写 H2，建立清晰的标题层级
4. ✅ 加粗带下划线高亮（模拟纸质报纸的重点标注）
5. ✅ 渐变分割线 + 米色引用块，营造印刷质感
6. ✅ 整体印象：读者第一眼想到的是《纽约时报》《财新》

## 四、排版转换执行清单

### 📋 前置准备
- [ ] 确认输入 Markdown 文件路径
- [ ] 检查是否需要先做结构化预处理（纯文本/缺少标题时）
- [ ] 确认主题选择：**`newspaper`**（纽约时报风格）

### 📋 Step 1: 内容结构化预处理（仅纯文本时需要）
- [ ] 统计 `##` 标题数量、列表数量、加粗数量
- [ ] 缺少结构标记时：
  - [ ] 在主题转换处插入 `##` 标题
  - [ ] 识别并列内容加 `-` 列表
  - [ ] 关键词/产品名加 `**加粗**`
  - [ ] 别人的话/原文摘抄用 `> 引用`
  - [ ] 保存为 `xxx-structured.md`
- [ ] 中英文/中数字间自动补空格
- [ ] 中文标点移出加粗标记外（`**文字**，` 而非 `**文字，**`）

### 📋 Step 2: 自定义容器增强
- [ ] 检测连续对话 → 套 `:::dialogue` 气泡
- [ ] 检测 3 张以上连续图 → 套 `:::gallery` 横向滚动
- [ ] 识别金句/核心观点 → 套 `[!important]` / `[!tip]` callout
- [ ] 图片后说明文字 → 斜体段
- [ ] 保存为 `xxx-enhanced.md`

### 📋 Step 3: 复杂排版降维（按需）
- [ ] 4 列以上宽表格 → HTML 绘制 → PNG 截图 → 原位替换
- [ ] 对比矩阵/流程图/架构图 → 同上处理
- [ ] ❌ 代码块/命令/URL **不要**图片化（读者需复制）
- [ ] PNG 宽度固定为 **1080px**（公众号配图标准）
- [ ] 源 HTML 保存在 `visuals/` 目录

### 📋 Step 4: 外链转脚注
- [ ] 正文所有非 `mp.weixin.qq.com` 的外链转为 `[1]` 角标
- [ ] 文末附参考链接列表
- [ ] 保留公众号自家链接和锚点链接

### 📋 Step 5: 执行渲染脚本
```bash
python3 skills/global/wechat-format-skill/bin/md2wechat.py \
  --input article-enhanced.md \
  --theme newspaper \
  --output article.html
```

### 📋 Step 6: 浏览器预览验证
- [ ] 复制输出的 `file://xxx.preview.html` URL
- [ ] 粘贴到浏览器地址栏 → 回车
- [ ] 检查以下项：
  - [ ] H1/H2/H3 样式正确（双线/居中大写）
  - [ ] 加粗带下划线高亮效果
  - [ ] 米色引用块样式
  - [ ] 列表圆点为 #326891 蓝色
  - [ ] 图片居中带圆角阴影
  - [ ] 对话气泡左右交替
  - [ ] 脚注格式正确

### 📋 Step 7: 接力到草稿（可选）
- [ ] 浏览器内 Cmd+A 全选 → Cmd+C 复制
- [ ] 公众号编辑器 Cmd+V 粘贴
- [ ] 或转交给 `wechat-mp-draft-skill` 自动化处理

## 五、反模式检查清单

- [ ] ❌ 不要使用花哨装饰（保持克制严肃）
- [ ] ❌ 不要混用无衬线字体（始终衬线优先）
- [ ] ❌ 不要过度使用 callout（一篇 1-3 处合理）
- [ ] ❌ 不要把作者自己的话包成引用（语义错乱）
- [ ] ❌ 不要跳过预览就交付（CSS 内联后才是真实效果）
- [ ] ❌ 不要用 `open` / 双击打开 HTML（必须从浏览器地址栏打开再复制）

## 六、脚本说明

渲染脚本位置：`skills/global/wechat-format-skill/bin/md2wechat.py`

脚本自动处理的兼容性问题：
- ✅ 所有样式内联（避免 class 被微信抹掉）
- ✅ `ul/ol` 自动转 section + flex（避免 padding/margin 被抹平）
- ✅ CJK 中英混排自动补空格
- ✅ 加粗内的中文标点自动移到外面
- ✅ 外链自动转脚注
- ✅ 首个 H1 自动删除（避免与公众号标题重复）
